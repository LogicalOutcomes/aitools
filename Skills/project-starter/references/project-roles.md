# Project Roles

These templates use three core project roles to ensure clear accountability and smooth delivery. Adapt role titles, responsibilities, and compensation details to match your organization's structure and policies.

## Project Owner

The Project Owner has overall responsibility for the project, including budget, schedule, and quality. They report directly to the Project Sponsor and supervise the work of the Project Manager.

### Responsibilities

- Monitor project expenditures based on weekly reports from the Project Manager
- Ensure work is carried out successfully according to the Project Charter terms
- Make revisions in scope or changes in team assignments as needed
- Absorb cost overruns so the organization does not incur losses on behalf of the project

### Compensation

The Project Owner generally charges a higher hourly rate to reflect the financial risk. They may need to work additional unpaid hours or pay out some of their earnings to other contractors to satisfactorily complete the project.

### Escalation

If the Project Sponsor (the client representative) has concerns about the project's progress, they should contact the appropriate senior person in your organization for troubleshooting. However, the Project Sponsor is responsible for monitoring progress and raising issues with the Project Owner and Project Manager as they arise.

Contractors and staff are not responsible for delivering a project unless they are named in the Project Charter and paid for their role. If clients wish the direct involvement of senior leadership, they must ensure these individuals are listed in the project team.

## Project Manager

Project Managers are responsible for defining, assigning, and monitoring the tasks of the project. They are supervised by the Project Owner and indirectly by the Project Sponsor.

### Responsibilities

- Define, assign, and monitor project tasks
- Report weekly on progress and costs incurred to date
- Post updates in the project tracking systems
- Report emerging risks, blockages, or issues so all team members can see and address them
- Report project risks to the President and Executive Director
- Ensure tasks are being carried out successfully

### Escalation

When Project Managers run into problems (e.g., a contractor ignores reminders or misses deadlines), they must:

1. Communicate with the Project Owner first
2. If that doesn't work, escalate to the Project Sponsor

Project Managers are encouraged to seek advice from senior leadership. However, senior leaders are not responsible for project quality unless they are named in the project team and paid for their role.

## Project Team Member

Team Members execute the work defined in the Project Charter. They:

- Receive task assignments from the Project Manager
- Report progress and blockers to the Project Manager
- Follow the project work plan and team communication protocols

Team Members will be notified of any changes to the Project Charter. No changes to scope or budget will be made without discussion first.
