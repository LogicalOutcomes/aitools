# Image Analysis and Provenance

Read this file only when an image is actually in play. The high-level trigger — describe, transcribe, and ask the user to confirm before continuing — lives in SKILL.md; this file is the detailed method.

Adapted from Mike Caulfield's Deep Background / SIFT Toolbox prompt: https://checkplease.neocities.org/sift-toolbox/index.html

---

## Method

1. Describe the visual elements objectively first, without commenting on meaning or underlying reality. Transcribe all text. Hedge honestly if something is not clearly visible. **Ask the user to correct the description/transcription before continuing.**
2. Verify dates, locations, and identities. When a photo is uploaded, search Alamy, Getty, and Granger archives for well-captioned versions.
3. Assess for signs of manipulation or mislabelling.
4. Compare with verified historical photos when possible; link any photo match and encourage the user to verify it visually. Real images may be colourized, cropped, or otherwise altered — look for originals. Search for black-and-white versions of colour photos in case of colourization.
5. Consider contextual clues within the image (landscape, clothing, technology, signage, etc.).
6. For provenance, link as directly as possible to the original version, professionally captioned or archived.

A good image summary leads with provenance, then covers how people have reacted to and interpreted the object, the context needed for a more informed reaction or a deeper story, and paths for further exploration or action.

## Comparing two photos that may be the same

1. Describe both photos in detail to yourself — objects, number of people, colours visible, photo style.
2. Print a basic summary of both.
3. Explicitly ask whether this is the same photo or a different one.
